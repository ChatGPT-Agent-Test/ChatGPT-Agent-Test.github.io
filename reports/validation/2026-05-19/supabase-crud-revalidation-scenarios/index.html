시나리오

단계 목표 기대 결과
1. 연결 확인 프로젝트 상태와 대상 테이블 구조 조회 프로젝트가 ACTIVE_HEALTHY 상태이고 4개 테이블이 존재해야 함
2. 테스트 사용자 준비 auth.users와 public.profiles에 검증용 사용자 생성 프로필 insert 성공, 후속 FK 경로 확보
3. profiles 조회/수정 삽입 직후 조회와 display_name 수정 조회 성공, updated_at 증가
4. reports 생성 고유 슬러그 기준 검증용 보고서 row insert insert 성공, report_id 확보
5. reports 조회/수정 status, summary 수정 후 재조회 update 성공, updated_at 증가
6. report_confirmations insert 초기 확인 row 생성 confirmation_count = 1
7. report_confirmations upsert 동일 키 재확인 처리 first_confirmed_at 유지, last_confirmed_at 증가, confirmation_count = 2
8. report_events 기록 재확인 이벤트 insert 및 조회 event row 생성 성공
9. 테스트 데이터 정리 events, confirmations, reports, profiles, auth.users 순서로 삭제 잔여 row 수가 모두 0이어야 함

주의 사항

public.profiles는 auth.users를 참조하므로, 이번 검증에서는 FK를 만족시키기 위해 테스트용 auth 사용자도 함께 만들고 마지막에 같이 정리합니다.

report_events에는 updated_at 컬럼이 없으므로 trigger 반영 검증 대상은 profiles, reports, report_confirmations로 제한합니다.
